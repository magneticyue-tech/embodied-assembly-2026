This is a PPT答辩 (oral defense) competition where language quality is judged. The user holds expression to a high standard.

**Why:** Words go onto slides / get spoken to judges; hedging, meta-commentary, and mixed value judgments read as unprofessional.

**How to apply:**
- Be concise and direct. Cut hedging and meta-narration (e.g. drop "先说结论…原因后面讲", "我猜…如果不对你纠正我").
- Avoid mixing value judgments with substance. Do NOT label content like "这是你真正的得分点"; just present it neutrally as "初步思路介绍".
- Use clean, neutral section headers: "对题目的理解", "初步思路介绍" — not editorializing titles like "这道题在考什么(给评审看的我读懂了)".
- At this stage, keep value judgments (what's most important / what wins) SEPARATE from the substance (the understanding and the proposal). Don't blend them.
- Write in Chinese for this project.

**素材核对/分析输出标准（长期）:** 遵循「事实先行、推论后置、边界保留、用词克制」。
- 事实先行：先客观陈述文档/资料中实际存在的内容，再给推论或建议。
- 区分观测与推断：把"文档里有什么"与"由此推断/归类"分开陈述，不混为一谈。
- 禁止无统计支撑的绝对化结论（如"基本100%""本质仍是X"）；用客观描述替代主观定性。
- 缺失素材单独列为一类事实（"文档暂未包含X"），不强行全盘归类、不一刀切。
- 明确区分「现有可复用素材」与「需全新补充素材」，标注来源。
- 不制造片面/绝对化结论迎合阅读偏好。
- 反例(CLAUDE.md):标题写成「仿真边界(答辩要诚实说明)」「关键事实(容易搞错,先记住)」「进度(四阶段)」——括号内的价值判断须去掉,只留中性标题「仿真边界」「关键事实」「进度」;正文同理,事实与价值判断分离。搬运源文档(如 README)措辞时,先套用本规则再落笔。
