# 仿真演示 (sim/)

用纯软件复现赛题完整闭环, **无需真机、无需 GPU**, 产出标注图 + 带时间戳日志, 可直接用于答辩。

## 它演示了什么

完整具身智能闭环 (赛题全流程):
```
语音唤醒「小具同学」 -> 任务卡识别(VLM替身) -> 语音复述
   -> [闭环装配6块: 每块前重新拍照定位] -> 「任务已完成」-> 「两个任务均已完成」
```

**用真算法, 不是动画**:
- `vision.py`: 真实 OpenCV 管线 (BGR->HSV 分割 -> 形态学 -> 轮廓 -> minAreaRect -> 单应反投影到台面 mm)
- **旋转鲁棒**: 方块/托盘可旋转, 用 minAreaRect 取中心+角度 (非外接正框); 正方形 90° 对称折叠到 [-45,45]
- **整块刚性托盘板 + 遮挡鲁棒** (本版重点):
  - 托盘是**一整块矩形板**, 板上 6 个同心环色槽按固定布局排布, 共享板的位姿。
  - **颜色→槽位映射**: 开局无遮挡时**一次性标定**, 之后固定不变。
  - **板位姿**: 每步靠**板外轮廓矩形**重新检测 (整块板永不被小方块遮挡; 中途裁判挪板也能跟上)。
  - 装配时**目标槽位置 = 板位姿 ⊕ 标定布局**, 不靠重新识别颜色 ——
    所以方块放上去盖住槽色、或放置方块与其它槽同色, 都不会造成混淆。
- **同心环精度尺**: 槽按 troy.png 渲染 (每环 1mm, 每 5mm 加粗, ±20mm), 放置偏移按"落在第几环"读出
- `scene.perturb()`: 每步前模拟裁判挪动/转动自由方块 **+ 整块托盘板**, 验证"闭环再感知"
- 逐步高亮当前方块(红框)与目标槽(黄框), 满足评分细则"界面高亮当前执行步骤"

## 运行

```bash
cd sim
python main.py
```
依赖: `numpy opencv-python pillow` (已验证可在 Python 3.14 安装)。
缺包时: `python -m pip install numpy opencv-python pillow`

| 文件 | 对应方案章节 |

## 仿真的边界 (答辩时要诚实说明)

- **是**: 验证视觉/认知/调度逻辑的正确性, 端到端跑通, 产出可视化与日志。
- **不是**: 物理动力学仿真。吸盘吸附力、碰撞、关节动力学未建模
  (pybullet 在 Python 3.14 无预编译包; 真机用 **AUBO SDK + ARCS 仿真**)。
- 误差几乎全部来自视觉定位 (本仿真单应理想, 故 <0.5mm); AUBO-i5 自身 ±0.02mm。
  真实赛场视觉误差更大, 同心环精度尺正是用来读这个误差的。
- 真机迁移需补: 相机内参+手眼标定、运动规划/避障、真实 HSV 阈值现场标定、VLM 实际部署。

## 真机/进阶路径

- 运动学/抓放: **ARCS 或 AUBOPE** (官方 AUBO 仿真, VMware + Python 脚本, 轨迹记录回放)。
- 视觉: 可选官方 **Vision Master** (海康 VM, 无代码拖拽); 本项目选自写 OpenCV 以获得
  旋转鲁棒、同心环误差读数与全管线可控 (理由见思路介绍文档)。
- 认知层实测: 把 `cognition.py` 规则解析替换为真实本地 VLM 调用。
