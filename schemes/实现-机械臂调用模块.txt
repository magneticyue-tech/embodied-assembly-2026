二、需求与实现对照（事实）

已实现，覆盖需求四条（机械臂调用模块.txt）：
- 抓取物块：AuboRobot.pick / SimRobot.pick，走坐标变换与 UDP，对应"调用求解器调整位姿抓取"（当前 C 层为占位桩，未接逆运动学 SDK）。
- 放置物块：AuboRobot.place / SimRobot.place，同上。
- 异常"操作失败"：由 RobotExecutionError + UDP 响应校验覆盖。
- 依赖"调度模块提供目标坐标/手眼标定"：目标位姿由入参传入，坐标变换配置由 config/命令行提供。

需求中列出、实现暂无：
- "调整视野"：execution 层无对应接口。视野/眼坐标计算在"后端调度模块.txt"中定义（手眼坐标互转、抓取/装配视野），execution.py 仅做台面↔机器人平面仿射变换，不含相机高度/内参那套视野解算。
- "复位"：需求列出复位(输入 void，依赖 SDK)，Python 层与 C 驱动均无 RESET/复位指令，仅有 PICK/PLACE 两类。

实现中有、需求表未列：
- 逆变换 robot_to_table、UDP 幂等（request_id + 128 条缓存）、超时重试、真机安全门禁（REAL_PERCEPTION_READY / REAL_TRANSFORM_CALIBRATED 未就绪则拒绝创建 AuboRobot）。
- 仿真评分 RingEvaluator 与落环换算（属仿真验证，非真机调用链）。

测试覆盖（code/tests）：
- test_execution.py：pick 失败抛错、部分变换参数合并默认值、scale=0 被拒、重试复用同一 request_id、拒绝数值型 success。
- test_main.py：真机模式在感知/标定就绪前被门禁拦截、机械臂失败即安全停止不再放置、视觉目标缺失即停止。
- test_robot_driver_build.py：有 GCC 时以 -Wall -Wextra -Werror 编译并运行 C 驱动测试，无 GCC 标记跳过。



三、接线与推断（推断，需联调确认）

- 事实：C 驱动 robot_driver.c 头注释与 README 均标注为占位桩，PICK/PLACE 仅打印模拟运动日志，未接入 AUBO-i5 厂商 SDK，不执行逆运动学与真实运动。
- 推断：需求表"调用 SDK 逆运动学求解器调整位姿"这一步当前落在待完成项（README 待完成工作第 1 条），需接入真实 SDK 后才成立。
- 待确认：需求的"调整视野""复位"两项归属——是并入 execution 层新增接口，还是保留在后端调度模块，需在需求对齐时定夺。
- 待确认：坐标变换目前是台面↔机器人的 2D 仿射（x_offset/y_offset/rotation_deg/scale），与后端调度模块基于手眼标定矩阵+相机内参的视野解算是两套；二者如何衔接（谁产出最终机器人坐标）需明确。
