**工具链(来源:赛方资料):**

从任务书推导可落地的技术指标。

| 赛题原始要求 | 推导出的技术指标 / 约束 |

## 2. 系统总体架构

四层架构:交互层横跨认知、感知与执行层,提供语音、日志和过程可视化:

```
┌─────────────────────────────────────────────┐
│  交互层  语音播报 · 时间戳日志 · 过程可视化     │
└─────────────────────────────────────────────┘
┌─────────────────────────────────────────────┐
│  认知层  大模型智能体 Agent                    │
│  · 任务卡识别/场景理解  · 指令解析→结构化JSON    │
│  · 任务规划(装配顺序)   · 对话与播报话术生成     │
└───────────────┬─────────────────────────────┘
        指令/规划 ↓        ↑ 感知结果/状态
┌───────────────┴─────────────────────────────┐
│  感知层                                      │
│  · 视觉: 标定→拍照→颜色分割→定位→反投影         │
│  · 语音: 唤醒(KWS)→识别(ASR)→合成(TTS)        │
└───────────────┬─────────────────────────────┘
        目标世界坐标 ↓     ↑ 执行反馈
┌───────────────┴─────────────────────────────┐
│  执行层  机器人控制                            │
│  · 坐标变换  · 运动规划  · 吸盘抓放  · 顺序控制  │
└─────────────────────────────────────────────┘
```

**数据流(以任务二一次装配为例):**
1. 语音唤醒 → ASR 触发 → Agent 调用视觉读任务卡 → 输出结构化装配序列 JSON
2. Agent 语音复述指令 → 取序列第 1 步(如"红块→黄盘")
3. 执行层移至拍照点 → 视觉对当前场景定位红块与黄盘 → 反投影出世界坐标
4. 运动规划 → 吸盘抓红块 → 放至黄盘 → 反馈成功
5. 回到第 3 步处理下一块(每块重新感知,应对中途挪动)
6. 6 块完成 → 播报"任务已完成"

**模块间通信:** 视觉(Python/OpenCV)、Agent、机器人控制走本地进程通信(socket 或共享消息);当前执行层为 Python 接口 + UDP + C 驱动占位桩,真实 AUBO C SDK 接口待设备联调。

## 3. 感知层 · 视觉模块详细设计

视觉流程:标定 → 拍照 → 颜色识别 → 定位 → 反投影。

### 3.0 视觉平台选型:自写 OpenCV vs Vision Master

赛方提供海康 Vision Master(无代码拖拽式视觉平台)。本方案选自写 OpenCV,理由:算法细节可控(托盘遮挡策略、旋转歧义修正需要定制逻辑)、便于与 Python 主流程集成。代价:开发与调参工作量由团队承担,无平台图形化调试工具。「待确认」现场是否有 Vision Master 加密狗依赖问题。

### 3.1 相机标定

**相机内参标定**:用棋盘格标定板,OpenCV `calibrateCamera` 求内参矩阵 K 与畸变系数。
- K = [[fx, 0, cx], [0, fy, cy], [0, 0, 1]],fx/fy 为焦距(像素),cx/cy 为主点。

**手眼标定**:相机挂载在机械臂末端(eye-in-hand),需求出相机坐标系与机器人末端坐标系之间的固定变换 X(齐次变换矩阵 4×4)。
- 方法:控制机械臂以多个不同姿态拍同一标定板,采集多组(机器人末端位姿 A_i, 标定板相对相机位姿 B_i),求解 AX = XB(OpenCV `calibrateHandEye`)。
- 输出 X 后,任意时刻已知机器人末端位姿,即可把相机坐标系下的点变换到机器人基坐标系。

### 3.2 拍照策略

- 设置一个或多个固定「拍照点」(机器人位姿),保证台面物料在视野内、光照均匀。
- 每次装配前回到拍照点重新拍照(闭环再感知,应对裁判挪动)。
- 配合光源减少阴影与反光;曝光/白平衡固定,减少颜色漂移。

### 3.3 颜色分割与定位

**颜色分割**:
1. 转 HSV → 对每个颜色用 `inRange` 得到二值掩膜
2. 形态学开运算去噪 → 闭运算
3. `findContours` 找轮廓 → 按面积过滤
4. 对每个有效轮廓求最小外接旋转矩形 `minAreaRect` → 得中心 + 旋转角(图像坐标)

**旋转处理**:方块/托盘的边不一定与画面平行。用 minAreaRect 获取中心与朝向。正方形 90° 旋转对称,角度折叠到 [-45°,45°) 再使用/比较。

**方块 vs 托盘区分**:托盘中心也是实心彩色方块(被同心环包围),与方块中心同色同形。区分方法:在彩色方块外围环带统计边缘密度(Canny)——托盘外有同心灰环,边缘密度高;方块在空台面上,边缘密度低。30mm 方块在已知视距下的像素尺寸已知,可作面积约束。(实现:code/sim/vision.py)

### 3.4 像素 → 世界坐标(平面反投影)

**方法 A:单应矩阵 H(本方案主用)**
- 台面是一个平面,图像平面到台面平面之间存在单应变换 H(3×3)。
- 赛前在台面上放标定板或已知坐标的标记点,采集 ≥4 组「像素坐标 ↔ 台面物理坐标」,`findHomography` 求 H。
- 运行时:任意方块/托盘质心像素 (u,v) → 经 H 映射 → 台面物理坐标 (X,Y)。Z 为已知台面高度(+ 半个方块高度作为抓取点)。
- 选择理由:实现步骤少,标定一次即可;精度依赖标定点分布与台面平面假设成立。

**方法 B:已知尺寸 + PnP**
- 利用 30mm 方块的已知尺寸,对方块顶面四角解 PnP 求位姿。适用面更广,但对角点检测精度敏感。
- 作为方法 A 的补充或交叉校验。

**得到台面坐标后** → 经手眼变换 X 与当前末端位姿,转为机器人基坐标系下的抓取目标。

### 3.5 任务卡识别

任务卡内容交给多模态大模型(VLM):把任务卡照片上传 VLM,输出结构化结果(场景描述 / 装配序列 JSON)。

## 4. 感知层 · 语音模块详细设计

语音链路:唤醒(KWS)→ 识别(ASR)→ 合成(TTS)。

### 4.1 语音唤醒 KWS

- 唤醒词固定为「小具同学」,需可定制唤醒词的离线引擎。
- 候选:Picovoice Porcupine(可训自定义词)、科大讯飞离线唤醒 SDK 等。Snowboy 已停止维护,列为备选。「待确认」最终选型需实测唤醒率后确定。
- 唤醒后用 TTS 播报固定话术「我已就绪,请下达指令」。

### 4.2 语音识别 ASR

- 离线候选:FunASR(中文模型,可离线)、Whisper / faster-whisper(多语种,GPU 加速)、Vosk(轻量离线)。「待确认」选型需实测识别率后确定。
- 用途界定:已确认装配指令来自相机读卡(VLM 解析),不是裁判口述。ASR 只负责唤醒后的交互触发;装配指令解析不依赖 ASR。
- 输出文本送 Agent 解析。

### 4.3 语音合成 TTS

- 离线候选:Piper(离线)、PaddleSpeech TTS、edge-tts(需网络)、厂商离线 TTS SDK。「待确认」
- 需播报的固定话术:应答语、指令复述、"任务已完成"、"两个任务均已完成"、识别失败提示。

### 4.4 备用文本输入

- 规则要求必须具备文本输入功能,仅作语音失败时备用,使用前需向裁判申请,每条扣分。
- 实现为一个输入框,接同一套解析逻辑。规则禁止按键/触摸屏/串口等非指定方式——文本输入的具体合规形式需对照规则与培训确认。「待确认」

## 5. 认知层 · 大模型智能体详细设计

### 5.1 模型选型

当前候选:豆包多模态大模型(云端 API),一个模型承担任务卡识别、场景理解、指令解析,减少环节。语音唤醒/合成在本地运行,现场网络自备。「待确认」AI 边缘计算平台的算力上限;「待确认」云端 API 在自备网络下的时延与可用性,需实测后决定是否本地部署开源 VLM 作为备份。

### 5.2 结构化输出契约

大模型输出严格 JSON,执行层只认 JSON,避免自然语言歧义。示例(任务二):

```json
{
  "task": "assembly",
  "confidence": 0.97,
  "sequence": [
    {"step": 1, "block_color": "red",    "tray_color": "yellow"},
    {"step": 2, "block_color": "green",  "tray_color": "red"},
    {"step": 3, "block_color": "orange", "tray_color": "blue"},
    {"step": 4, "block_color": "blue",   "tray_color": "purple"},
    {"step": 5, "block_color": "yellow", "tray_color": "orange"},
    {"step": 6, "block_color": "purple", "tray_color": "green"}
  ]
}
```

- 用 prompt 约束输出格式;加 JSON Schema 校验,解析失败时播报规定提示并进入失败分支,按赛题要求重抽任务卡并重启流程。
- `confidence` 低于阈值 → 触发识别失败语音。

### 5.3 提示词工程要点

- 明确角色与任务、给出颜色枚举(红橙黄绿蓝紫)、强制 JSON 输出、给 few-shot 示例。
- 场景识别(任务一)和指令解析(任务二)用不同 prompt 模板。
- 要求模型对模糊/遮挡情况报告而非编造。

### 5.4 日志

解析过程必须以文本保存且带时间戳(赛题要求)。每次大模型调用记录:时间戳、输入(图片路径/文本)、输出 JSON、置信度。落盘为日志文件。

## 6. 执行层 · 机器人控制

### 6.1 坐标系与变换链

```
像素坐标 (u,v)
   │  内参 K + 畸变矫正
   ▼
归一化图像坐标
   │  单应 H(平面反投影,3.4 节)
   ▼
台面物理坐标 (X,Y),Z=台面高度
   │  相机坐标系下表示
   │  手眼变换 X(3.1 节)+ 当前末端位姿 T_base→ee
   ▼
机器人基坐标系下抓取目标位姿
```

### 6.2 抓取与放置

- 末端是电动吸盘,抓取正方体顶面中心。抓取点 = 方块质心的 (X,Y) + 顶面高度 Z。
- 角度对齐:用视觉给出的旋转角对齐末端,使方块边与托盘色框对齐。
- 抓取流程:移动到目标上方安全高度 → 垂直下降到接触高度 → 开吸盘 → 上抬 → 移动到托盘上方 → 下降到放置高度 → 关吸盘释放 → 上抬。
- 放置约束:方块须落入托盘对应颜色框、放后稳定不倾倒、不可二次触碰。放置高度和释放时机需精确控制,避免砸落或偏移。

### 6.3 运动规划

- 点位运动用 AUBO SDK 的关节/直线运动指令(MoveJ/MoveL)。
- 设置安全高度过渡点,避免碰撞已放置的方块和托盘。
- 路径中加入避让逻辑:已装配区域的方块不可碰。

### 6.4 双任务调度与状态机

任务一、二无先后,顺序随机。每次完成播报,全部完成总播报。状态机:

```
[空闲] --唤醒"小具同学"--> [就绪:播报应答]
   │
   ├─任务卡1--> [场景识别] --成功--> [播报"任务已完成"] ─┐
   │                       └失败--> [重抽/重启程序]      │
   │                                                     ▼
   ├─任务卡2--> [指令解析] --> [循环装配6块] --成功--> [播报"任务已完成"]
   │              每块: 拍照→定位→抓→放(闭环再感知)        │
   │                       └任一步失败--> [记分后重启]      │
   │                                                     ▼
   └──────────────> [两任务均完成?] --是--> [播报"两个任务均已完成"]
```

- 单次任务失败:按规则记分后执行另一个任务,重启程序。
- 5 分钟内可重抽任务卡重来,取最优成绩;程序需支持快速复位到初始状态。

## 7. 关键风险与对策

| 风险 | 对策 |
|---|---|
| 手眼/平面标定误差致抓偏 | 赛前反复标定+验证;现场保留快速校准流程;预留误差预算 |
| 裁判中途挪动方块/托盘 | 每块装配前重新拍照定位(闭环再感知),不依赖历史位置 |
| 相近色误判(红橙、蓝紫) | 受控光源;实测固化 HSV 阈值;留色相间隔;必要时多特征联合 |
| 现场无网导致云端模型不可用 | 自备网络实测;本地部署开源 VLM 作为备份「待确认」 |
| 大模型输出非法 JSON | Schema 校验+失败分支;播报规定提示后按规则重抽/重启;固定话术预生成 |
| 光照变化致颜色漂移 | 固定曝光/白平衡;光源遮挡环境光;赛前在现场光照下重标定 |
| 全自动流程中断 | 状态机管理;异常捕获;支持快速复位重来取最优 |
| 系统文件误删致机器人崩溃(判 0 分) | 备份;不动系统文件;按培训视频规范操作 |

## 8. 验证方案(无需真机可推进的部分)

| 模块 | 验证方式 |
|---|---|
| 任务卡识别 / 指令解析 | 拍真实任务卡照片(或自制样卡)喂 VLM,核对 JSON |
| 颜色分割 / 定位 | 拍彩色方块+托盘照片,跑 OpenCV 管线核对结果 |
| 平面反投影精度 | 用标定板照片,核对反投影坐标与真值误差 |
| 语音唤醒/ASR/TTS | 麦克风+音箱本地测,念唤醒词、测识别率 |
| 运动/抓放流程 | ARCS/AUBOPE 官方仿真跑通逻辑 |
| 视觉+认知+调度逻辑 | 本仓库 code/sim/(纯软件端到端;未通过人工评审,见 reviews/) |
| 全链路联调 | ARCS 仿真或真机端到端 |
