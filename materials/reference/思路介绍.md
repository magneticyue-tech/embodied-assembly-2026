## 5. 关键解决方案

【PPT要点】
- 5.1 装配前重定位
- 5.2 固定托盘颜色标定
- 5.5 安全冗余算法

针对三个难点,我们有对应的解决方案：我们在机械臂装配物块前重新拍照定位物块和托盘，保证装配精度；用托盘整体轮廓定位托盘，并根据初始情况，固定托盘颜色与相对位置的映射关系，避免装配完成的方块混淆后续的装配；同时，对每一步操作设计冗余路径容错、轻微扰动自动重试，提升系统的鲁棒性。

## 1. 总体架构: 四层闭环

【PPT要点】

    ┌──────────────────────────────────────────┐
    │ 交互层: 可视化界面 + 时间戳日志 + 语音/文本 │
    ├────────────┬───────────┬─────────────────┤
    │  感知层    │  认知层    │   执行层         │
    │ 相机+CV    │ Doubao    │  AUBO-i5        │
    │ 麦克风+KWS │ 智能体     │                 │
    └───────────┴────────────┴─────────────────┘

- 数据流 (任务二): 语音唤醒 → 读取任务卡 → 语音复述 → [定位方块→抓→定位托盘→放] ×6 → 完成播报。
- 数据流 (任务一): 语音唤醒 → 读取任务卡 → 场景识别 → 语音播报 → 完成播报。

> 🖼 图片: 四层架构图 (交互层横跨顶部, 感知/认知/执行三柱 + 数据流箭头)

【讲稿】
我们从四个层级设计了具身智能系统。感知层识别语音指令,用 OpenCV 管线识别并定位方块和托盘;认知层用云端多模态大模型 Doubao 读取、理解任务卡;交互层用离线 TTS 合成播报语音,并用可视化界面和时间戳日志展示全过程。系统在抓取前定位方块、放置前定位托盘,执行层的每个动作都基于最新画面,最大限度提高精度。

## 3. 感知层·视觉

【PPT要点】
- 管线: 图像分割 → 形态学去噪 → 轮廓 → **minAreaRect(中心+角度)** → 单应反映射到台面坐标。
//可选：生成模拟管线视频
- **旋转鲁棒**: 旋转矩形取朝向; 正方形 90° 对称折叠到 [-45°,45°]。

【讲稿】
我们已经使用OpenCV构建了感知层视觉管线。流程是:将图像按颜色分割, 识别物体轮廓，再通过相机内参矩阵映射到台面坐标。考虑到方块和托盘可能带有旋转角度，我们使用最小外接旋转矩形算法，获取方块中心和旋转角,增强系统对物体朝向的鲁棒性。

---

## 4. 认知层 · 大模型智能体

【PPT要点】
- 选型: **Doubao 多模态大模型**, 任务卡识别 + 场景理解 + 指令解析。
- **结构化输出**: JSON 格式化输出
图片/代码块示例:
  `{"step":1,"block_color":"red","tray_color":"yellow"}` ×6 + `confidence`。
- **置信度校验**: 低于阈值或识别模糊 →「重新识别」。

【讲稿】
在认知层我们使用Doubao大模型, 承担识别任务卡、理解场景、解析装配指令等认知任务。我们使用提示词工程强制JSON格式化输出,并在数据出口进行语法校验。同时，若文字识别置信度不够、任务卡模糊,大模型返回“重新识别”，系统进入容错路径，轻微扰动自动重试。

---

## 5. 执行层 · 机器人控制 (AUBO-i5)

【PPT要点】
- 坐标变换: 像素 →(内参矩阵)→ 台面坐标 →(手眼变换 + 末端位姿)→ AUBO-i5 基坐标抓取位姿。
- 吸盘抓放: 对准方块中心 + 旋转角对齐; 安全高度过渡
- 调试验证: 官方 **ARCS仿真系统**

【讲稿】
执行层是AUBO-i5机械臂。我们通过一系列坐标变换，由图片像素坐标得到机械臂的姿态参数；吸盘按物块旋转角对齐,在安全高度进行搬运、避开已放好的方块。赛前我们用官方的ARCS仿真系统完整实现整套运动，提升系统确定性。

## 6. 交互层

【PPT要点】
- **视觉处理可视化**
- **大模型推理可视化**
- **装配过程可观测**
图片:一个四联屏监控界面
    ┌──────────────────┬──────────────────┐
    │ ① 视觉处理        │ ② 大模型推理日志   │
    │   (实时画面+叠加)  │   (时间戳滚动)     │
    ├──────────────────┼──────────────────┤
    │ ③ 装配进度        │ ④ 设备/系统状态    │
    │   (步骤高亮)       │   (状态机+机械臂)  │
    └──────────────────┴──────────────────┘
    ① 视觉处理可视化

    接入工业相机视频流,实时叠加处理结果。在展示图片中文字标注：
    - 方块旋转框(minAreaRect)+ 中心点 + 台面坐标(mm)+ 旋转角
    - 托盘板外轮廓框 + 各槽位标记
    - 可选：小窗显示 HSV 掩膜图。
    ② 大模型推理可视化

    一个带时间戳的滚动日志面板(赛题强制项)。包括:
    - 时间戳
    - 输入(任务卡缩略图/路径)
    - 输出(解析出的 JSON 指令序列)
    - 置信度

    同时存档,满足"解析过程保存带时间戳文本"的要求。 
    
    ③ 装配过程可观测 + 设备状态

    - 语音播报当前动作:"正在执行第 3 步,抓取橙色方块,放到蓝色托盘"
    - 界面高亮当前步骤:步骤列表/进度条 + 当前方块红框、目标槽黄框,已完成 X/6
    - 设备运行状态栏:机械臂状态(就绪/移动中/抓取中)、当前状态机所处状态

【讲稿】
我们针对可视化的评分细则设计了交互层监控界面。视觉处理方面，我们接入相机视频流，实时显示图像处理结果；大模型推理方面，我们使用带时间戳的滚动日志面板记录大模型调用。装配过程方面，我们在设备状态栏高亮显示当前步骤，同时进行语音播报。完整的交互界面将极大提升人机交互的沟通效率，提高系统的可解释性。
