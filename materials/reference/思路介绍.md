## 5. 关键解决方案

【PPT要点】
- 5.1 装配前重定位
- 5.2 固定托盘颜色标定
- 5.5 安全冗余算法

针对三个难点,我们有对应的解决方案：机械臂装配物块前重新拍照定位物块和托盘；用托盘整体轮廓定位托盘，并根据初始情况固定托盘颜色与相对位置的映射关系，避免装配完成的方块混淆后续装配；执行中任一步失败时安全停止，并按赛题要求申请重抽任务卡、重新启动流程。

## 1. 总体架构: 四层闭环

【PPT要点】

    ┌──────────────────────────────────────────┐
    │ 交互层: 可视化界面 + 时间戳日志 + 语音/文本 │
    ├────────────┬───────────┬─────────────────┤
    │  感知层    │  认知层    │   执行层         │
    │ 相机+CV    │ Doubao    │  AUBO-i5        │
    │ 麦克风+KWS │ 智能体     │                 │
    └───────────┴────────────┴─────────────────┘

- 数据流 (任务二): 语音唤醒 → 读取任务卡 → 语音复述 → [定位方块→抓→定位托盘→放] ×6 → 完成播报。
- 数据流 (任务一): 语音唤醒 → 读取任务卡 → 场景识别 → 语音播报 → 完成播报。

> 🖼 图片: 四层架构图 (交互层横跨顶部, 感知/认知/执行三柱 + 数据流箭头)

【讲稿】
我们从四个层级设计具身智能系统。感知层计划接入语音与工业相机,并用 OpenCV 管线识别、定位方块和托盘;认知层候选使用 Doubao 多模态模型读取任务卡;交互层计划使用离线 TTS、可视化界面和时间戳日志。当前软件仿真使用合成画面、规则认知和控制台交互桩,在每次装配前重新感知场景。

## 3. 感知层·视觉

【PPT要点】
- 管线: 图像分割 → 形态学去噪 → 轮廓 → **minAreaRect(中心+角度)** → 单应反映射到台面坐标。
//可选：生成模拟管线视频
- **旋转鲁棒**: 旋转矩形取朝向; 正方形 90° 对称折叠到 [-45°,45°]。

【讲稿】
我们已经使用 OpenCV 构建了软件仿真视觉管线。流程是:将图像按颜色分割,识别物体轮廓,再通过平面单应变换映射到台面坐标。考虑到方块和托盘可能带有旋转角度,使用最小外接旋转矩形算法获取方块中心和旋转角。真实相机标定与定位精度仍需实测。

---

## 4. 认知层 · 大模型智能体

【PPT要点】
- 选型: **Doubao 多模态大模型**, 任务卡识别 + 场景理解 + 指令解析。
- **结构化输出**: JSON 格式化输出
图片/代码块示例:
  `{"step":1,"block_color":"red","tray_color":"yellow"}` ×6 + `confidence`。
- **置信度校验**: 低于阈值或识别模糊 →「重新识别」。

【讲稿】
认知层计划使用 Doubao 多模态模型承担任务卡识别、场景理解和装配指令解析。模型输出采用 JSON 格式并在数据出口进行 Schema 校验；若置信度不足、任务卡模糊或输出无效，系统播报规定的识别失败提示并停止，按赛题要求重抽任务卡、重启流程。当前 main 仍使用规则桩，真实模型接口待集成。

---

## 5. 执行层 · 机器人控制 (AUBO-i5)

【PPT要点】
- 坐标变换: 像素 →(内参矩阵)→ 台面坐标 →(手眼变换 + 末端位姿)→ AUBO-i5 基坐标抓取位姿。
- 吸盘抓放: 对准方块中心 + 旋转角对齐; 安全高度过渡
- 调试验证: 官方 **ARCS仿真系统**

【讲稿】
执行层目标设备是 AUBO-i5 机械臂。系统计划通过相机标定、平面反投影和手眼变换得到机器人目标位姿,并在安全高度搬运。当前仓库已实现 Python 接口、坐标变换和 C 驱动占位桩；真实 SDK 与 ARCS/设备运动仍待联调。

## 6. 交互层

【PPT要点】
- **视觉处理可视化**
- **大模型推理可视化**
- **装配过程可观测**
图片:一个四联屏监控界面
    ┌──────────────────┬──────────────────┐
    │ ① 视觉处理        │ ② 大模型推理日志   │
    │   (实时画面+叠加)  │   (时间戳滚动)     │
    ├──────────────────┼──────────────────┤
    │ ③ 装配进度        │ ④ 设备/系统状态    │
    │   (步骤高亮)       │   (状态机+机械臂)  │
    └──────────────────┴──────────────────┘
    ① 视觉处理可视化

    接入工业相机视频流,实时叠加处理结果。在展示图片中文字标注：
    - 方块旋转框(minAreaRect)+ 中心点 + 台面坐标(mm)+ 旋转角
    - 托盘板外轮廓框 + 各槽位标记
    - 可选：小窗显示 HSV 掩膜图。
    ② 大模型推理可视化

    一个带时间戳的滚动日志面板(赛题强制项)。包括:
    - 时间戳
    - 输入(任务卡缩略图/路径)
    - 输出(解析出的 JSON 指令序列)
    - 置信度

    同时存档,满足"解析过程保存带时间戳文本"的要求。 
    
    ③ 装配过程可观测 + 设备状态

    - 语音播报当前动作:"正在执行第 3 步,抓取橙色方块,放到蓝色托盘"
    - 界面高亮当前步骤:步骤列表/进度条 + 当前方块红框、目标槽黄框,已完成 X/6
    - 设备运行状态栏:机械臂状态(就绪/移动中/抓取中)、当前状态机所处状态

【讲稿】
交互层监控界面需要覆盖评分细则中的三类可观测内容:视觉处理结果、大模型带时间戳日志和装配步骤。当前仓库已生成视觉标注图与时间戳文本日志,图形界面、相机视频流、真实模型调用和设备状态仍待实现。
